# .env 파일에 Oracle Instant Client 경로 추가 가이드

## 📝 단계별 설명

### 1단계: .env 파일 위치 확인

`.env` 파일은 **프로젝트 루트 디렉토리**에 있어야 합니다.

```
C:\Users\134\Desktop\DX Project\
  ├── .env          ← 여기!
  ├── manage.py
  ├── config\
  ├── api\
  └── ...
```

---

### 2단계: .env 파일 찾기

#### 방법 1: VS Code에서

1. **VS Code 열기**
2. **프로젝트 폴더 열기**: `C:\Users\134\Desktop\DX Project`
3. **파일 탐색기에서 `.env` 파일 찾기**
   - 파일 이름이 `.env`로 시작하는 파일
   - 보이지 않으면 숨김 파일 표시 설정 확인

#### 방법 2: 파일 탐색기에서

1. **파일 탐색기 열기**
2. 다음 경로로 이동:
   ```
   C:\Users\134\Desktop\DX Project
   ```
3. **파일 보기 설정 변경**
   - 상단 메뉴: **보기** → **숨긴 항목** 체크
   - 이제 `.env` 파일이 보일 것입니다

#### 방법 3: PowerShell에서

```powershell
cd "C:\Users\134\Desktop\DX Project"
notepad .env
```

이 명령으로 `.env` 파일을 메모장으로 열 수 있습니다.

---

### 3단계: .env 파일 열기

#### 방법 1: VS Code에서 열기
- 파일 탐색기에서 `.env` 파일을 **더블클릭** 또는 **우클릭 → 열기**

#### 방법 2: 메모장으로 열기
- 파일 탐색기에서 `.env` 파일을 **우클릭 → 메모장으로 열기**

#### 방법 3: PowerShell 명령어
```powershell
notepad .env
```

---

### 4단계: 경로 추가하기

`.env` 파일을 열었으면, 파일 끝에 다음 줄을 추가하세요:

```env
ORACLE_INSTANT_CLIENT_PATH=C:\oraclexe\instantclient-basic-windows.x64-23.26.0.0.0
```

**중요 사항**:
- ✅ `=` 앞뒤로 공백 없이
- ✅ 경로에 공백이 있으면 따옴표 없이 그대로 작성
- ✅ 맨 앞에 `#` 없이 (주석이 아님)

---

### 5단계: 예시 (전체 .env 파일)

`.env` 파일이 이렇게 생겼을 것입니다:


**마지막 줄**에 `ORACLE_INSTANT_CLIENT_PATH`를 추가하세요!

## ⚠️ 주의사항

### 1. 파일 형식
- ✅ 텍스트 파일이어야 합니다
- ❌ Word 문서(.docx)나 다른 형식이 아닙니다
- ✅ 메모장, VS Code, Notepad++ 등으로 열기

### 2. 경로 형식
```env
# ✅ 올바른 형식
ORACLE_INSTANT_CLIENT_PATH=C:\oraclexe\instantclient-basic-windows.x64-23.26.0.0.0

# ❌ 잘못된 형식
ORACLE_INSTANT_CLIENT_PATH = C:\oraclexe\...  # 공백 있음
ORACLE_INSTANT_CLIENT_PATH="C:\oraclexe\..."  # 따옴표 있음
# ORACLE_INSTANT_CLIENT_PATH=...  # 주석 처리됨
```

### 3. 경로에 공백이 있는 경우
경로에 공백이 있어도 따옴표로 감싸지 마세요:
```env
# ✅ 이렇게
ORACLE_INSTANT_CLIENT_PATH=C:\Program Files\oracle\instantclient

# ❌ 이렇게 하지 마세요
ORACLE_INSTANT_CLIENT_PATH="C:\Program Files\oracle\instantclient"
```

## 🔍 문제 해결

### 문제 1: .env 파일이 없어요
**해결**: `env.example` 파일을 복사하여 `.env` 파일 만들기
```powershell
Copy-Item env.example .env
```

### 문제 2: 경로가 작동하지 않아요
**확인 사항**:
1. 경로에 `oci.dll` 파일이 있는지 확인
2. 경로에 오타가 없는지 확인
3. 경로에 한글이나 특수문자가 있는지 확인

### 문제 3: 파일이 저장이 안 돼요
**해결**: 
- 관리자 권한으로 프로그램 실행
- 파일 속성에서 "읽기 전용" 해제
