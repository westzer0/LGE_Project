# 깨진 커밋 메시지 수정 가이드

## 🔍 문제 상황

커밋 `57880e5`의 메시지가 인코딩 문제로 깨졌습니다:
```
feat: 移댁뭅?ㅽ넚 怨듭쑀, AI 梨쀫큸, API ???ㅼ젙 湲곕뒫 異붽?
```

원래 의도된 메시지는:
```
feat: 카카오톡 공유, AI 챗봇, API 키 관리 기능 추가
```

## 🔧 수정 방법

### 옵션 1: 커밋 메시지 수정 (로컬만)

```bash
# 최근 커밋 메시지 수정
git commit --amend -m "feat: 카카오톡 공유, AI 챗봇, API 키 관리 기능 추가"

# 강제 푸시 (주의!)
git push origin main --force
```

⚠️ **주의**: 이미 푸시된 커밋을 수정하면 히스토리가 바뀝니다. 
팀원과 협업 중이라면 반드시 사전에 알려야 합니다!

### 옵션 2: 새 커밋으로 메시지 추가 (안전)

```bash
# 커밋 메시지만 수정하는 새 커밋 추가
git commit --allow-empty -m "docs: 커밋 메시지 인코딩 문제 해결

- 커밋 57880e5: 카카오톡 공유, AI 챗봇, API 키 관리 기능 추가"
```

### 옵션 3: 그냥 두기 (권장)

이미 다음 커밋(`339d212`)에서 영어로 명확하게 커밋했으므로,
깨진 메시지를 그대로 두어도 무방합니다.

## 📝 참고사항

PowerShell에서 한글 커밋 메시지를 작성할 때는:

1. **UTF-8 인코딩 사용**:
```bash
$env:PYTHONIOENCODING="utf-8"
git commit -m "한글 메시지"
```

2. **메시지 파일 사용**:
```bash
# message.txt 파일에 한글 작성 후
git commit -F message.txt
```

3. **영어 사용** (가장 안전):
```bash
git commit -m "feat: Add Kakao share, AI chatbot, API key management"
```

## ✅ 추천 방안

**현재 상황에서는 옵션 3을 권장합니다:**
- 이미 다음 커밋에서 영어로 명확하게 작성됨
- 히스토리 재작성의 위험 없음
- 깨진 메시지가 있어도 실제 코드는 정상

만약 반드시 수정하고 싶다면, **옵션 2(빈 커밋 추가)**를 사용하는 것이 가장 안전합니다.
