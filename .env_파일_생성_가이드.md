# .env 파일 생성 가이드

## 🔍 문제 발견

환경 변수가 모두 `None`으로 나오는 이유는 **`.env` 파일이 없거나 올바르게 설정되지 않았기 때문**입니다.

---

## ✅ 해결 방법

### 방법 1: env.example을 복사하여 .env 파일 만들기

**PowerShell에서 실행**:
```powershell
Copy-Item env.example .env
```

**또는 CMD에서**:
```cmd
copy env.example .env
```

### 방법 2: 직접 .env 파일 만들기

프로젝트 루트 디렉토리에 `.env` 파일을 만들고 다음 내용을 복사하세요:

```env
# LG 홈스타일링 프로젝트 환경 변수 설정

# Django Secret Key (필수)
DJANGO_SECRET_KEY=your_django_secret_key_here

# 카카오 API 키
KAKAO_REST_API_KEY=your_kakao_rest_api_key_here
KAKAO_JS_KEY=your_kakao_js_key_here

# OpenAI API 키
OPENAI_API_KEY=your_openai_api_key_here

# Oracle 데이터베이스 설정 (필수 - 아래 값을 실제 값으로 변경하세요)
ORACLE_USER=campus_24K_LG3_DX7_p3_4
ORACLE_PASSWORD=smhrd4
ORACLE_HOST=project-db-campus.smhrd.com
ORACLE_PORT=1524
ORACLE_SERVICE_NAME=MAPPP

# 기존 형식 (호환성 유지)
DB_NAME=MAPPP
DB_USER=campus_24K_LG3_DX7_p3_4
DB_PASSWORD=smhrd4
DB_HOST=project-db-campus.smhrd.com
DB_PORT=1524
DB_SERVICE_NAME=MAPPP
```

**중요**: 
- ✅ `ORACLE_PASSWORD` 값을 실제 비밀번호로 변경하세요
- ✅ `your_kakao_rest_api_key_here` 등을 실제 API 키로 변경하세요

---

## 🧪 .env 파일 확인

`.env` 파일을 만든 후, 다음 명령으로 확인하세요:

```powershell
python -c "import os; from dotenv import load_dotenv; from pathlib import Path; env_path = Path('.env'); load_dotenv(env_path); print('USER:', os.getenv('ORACLE_USER')); print('HOST:', os.getenv('ORACLE_HOST')); print('SERVICE_NAME:', os.getenv('ORACLE_SERVICE_NAME'))"
```

**예상 출력 (성공 시)**:
```
USER: campus_24K_LG3_DX7_p3_4
HOST: project-db-campus.smhrd.com
SERVICE_NAME: MAPPP
```

여전히 `None`이 나오면:
- `.env` 파일이 프로젝트 루트에 있는지 확인
- 파일 이름이 정확히 `.env`인지 확인 (`.env.txt` 아님)
- 파일 내용에 `ORACLE_USER=...` 형식으로 키가 있는지 확인

---

## 📝 주의사항

1. **`.env` 파일은 `.gitignore`에 포함되어 있어야 합니다**
   - Git에 커밋하지 마세요 (비밀번호가 포함되어 있음)

2. **파일 위치**
   - 프로젝트 루트 디렉토리에 있어야 합니다
   - `manage.py`와 같은 디렉토리에 있어야 합니다

3. **파일 이름**
   - 정확히 `.env` (점으로 시작)
   - `.env.txt` 또는 `env.txt` 아님

## 🔧 빠른 체크 스크립트

다음 Python 스크립트로 `.env` 파일 상태를 확인할 수 있습니다:

```python
from pathlib import Path
from dotenv import load_dotenv
import os

env_path = Path('.env')
print(f".env 파일 존재: {env_path.exists()}")
print(f".env 파일 경로: {env_path.resolve()}")

if env_path.exists():
    load_dotenv(env_path)
    print(f"\n환경 변수 로드:")
    print(f"  ORACLE_USER: {os.getenv('ORACLE_USER')}")
    print(f"  ORACLE_HOST: {os.getenv('ORACLE_HOST')}")
    print(f"  ORACLE_SERVICE_NAME: {os.getenv('ORACLE_SERVICE_NAME')}")
else:
    print("\n❌ .env 파일이 없습니다!")
    print("env.example 파일을 복사하여 .env 파일을 만드세요:")
    print("  Copy-Item env.example .env")
```
