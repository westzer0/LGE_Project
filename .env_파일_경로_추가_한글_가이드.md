# .env 파일에 경로 추가하는 방법 (초보자용)

## 📂 1단계: .env 파일 찾기

### VS Code나 파일 탐색기에서
1. 프로젝트 폴더 열기: `C:\Users\134\Desktop\DX Project`
2. `.env` 파일 찾기 (파일 이름이 `.env`)
3. 파일이 안 보이면:
   - **VS Code**: 왼쪽 파일 목록에서 찾기
   - **파일 탐색기**: 상단 메뉴 → **보기** → **숨긴 항목** 체크

---

## ✏️ 2단계: .env 파일 열기

### 방법 1: VS Code에서
- `.env` 파일을 **더블클릭**하여 열기

### 방법 2: 메모장에서
- `.env` 파일을 **우클릭** → **연결 프로그램** → **메모장**

### 방법 3: PowerShell 명령어
```powershell
notepad .env
```

---

## 📝 3단계: 경로 추가하기

### .env 파일 내용 확인

파일을 열면 대략 이런 내용이 보일 것입니다:

```
ORACLE_USER=campus_24K_LG3_DX7_p3_4
ORACLE_PASSWORD=smhrd4
ORACLE_HOST=project-db-campus.smhrd.com
ORACLE_PORT=1524
ORACLE_SID=xe
```

### 경로 추가하기

파일의 **맨 끝**에 다음 줄을 추가하세요:

```
ORACLE_INSTANT_CLIENT_PATH=C:\oraclexe\instantclient-basic-windows.x64-23.26.0.0.0
```

**예시 (추가 후)**:

## ✅ 5단계: 확인하기


또는:


## 🆘 문제 해결

### Q1: .env 파일이 없어요
**해결**:
```powershell
Copy-Item env.example .env
notepad .env
```
그 다음 위의 3단계부터 진행

### Q2: 파일이 저장이 안 돼요
**해결**: 
- 파일을 **우클릭** → **속성** → **읽기 전용** 체크 해제
- 또는 관리자 권한으로 프로그램 실행

### Q3: 경로에 오타가 있는 것 같아요
**확인 사항**:
- `=` 앞뒤로 공백 없이
- 경로가 정확한지 확인
- 대소문자 구분 확인 (Windows는 보통 대소문자 구분 안 함)
