# .env 파일 25번째 줄 파싱 오류 해결

## ⚠️ 경고 메시지

```
python-dotenv could not parse statement starting at line 25
```

이 경고는 `.env` 파일의 25번째 줄 근처에 형식이 잘못된 줄이 있다는 의미입니다.

---

## 🔍 확인 방법

### 1단계: .env 파일 열기

```powershell
notepad .env
```

### 2단계: 25번째 줄 확인

파일을 열어서 **25번째 줄 근처**를 확인하세요.

---

## ❌ 잘못된 형식 예시

### 1. `=` 기호가 빠진 경우
```
ORACLE_INSTANT_CLIENT_PATH C:\oracle\...   ← = 없음 (잘못됨)
```

**올바른 형식**:
```
ORACLE_INSTANT_CLIENT_PATH=C:\oracle\...   ← = 있음 (올바름)
```

### 2. 공백만 있는 줄에 이상한 문자
```
                                    ← 공백만 있고 이상한 문자가 있을 수 있음
```

**해결**: 이런 줄은 삭제하거나 주석 처리:
```
#                                    ← 주석 처리
```

### 3. 따옴표가 잘못 사용된 경우
```
ORACLE_INSTANT_CLIENT_PATH="C:\oracle\..."  ← 불필요한 따옴표 (경로에만 있으면 OK)
```

**일반적으로**:
- 경로에 공백이 없으면 따옴표 불필요
- 경로에 공백이 있어도 따옴표 없이 사용 가능

### 4. 특수 문자가 잘못된 경우
```
ORACLE_INSTANT_CLIENT_PATH=C:\oracle\...\    ← 끝에 백슬래시
```

**해결**: 끝에 백슬래시 제거

## 🔧 수정 방법

### 방법 1: 직접 수정
1. `.env` 파일 열기
2. 25번째 줄 근처 확인
3. 잘못된 줄 수정 또는 삭제

### 방법 2: 문제 있는 줄 찾기

PowerShell에서:
```powershell
Get-Content .env | Select-String -Pattern "^\s*[^#=]+\s+[^=]" | Select-Object LineNumber, Line
```

이 명령이 문제가 될 수 있는 줄을 찾아줍니다.

## 💡 빠른 해결

경고는 오류가 아니므로 무시해도 되지만, 깔끔하게 하려면:

1. **25번째 줄 근처 확인**
2. **형식이 맞지 않는 줄 수정 또는 삭제**
3. **모든 키=값 쌍이 올바른 형식인지 확인**
