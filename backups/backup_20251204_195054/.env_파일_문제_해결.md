# .env 파일 문제 해결

## 🔍 발견된 문제

터미널 출력:
```
USER: None
HOST: None
SERVICE_NAME: None
```

**원인**: `.env` 파일이 없거나 환경 변수가 로드되지 않았습니다.

---

## ✅ 즉시 해결 방법

### 1단계: .env 파일 생성

**PowerShell에서 실행**:
```powershell
Copy-Item env.example .env
```

**또는 수동으로**:
1. 프로젝트 루트 디렉토리에서 `env.example` 파일 복사
2. 파일 이름을 `.env`로 변경

### 2단계: .env 파일 편집

`.env` 파일을 열어서 다음 부분을 실제 값으로 변경하세요:

```env
# Oracle 데이터베이스 설정
ORACLE_USER=campus_24K_LG3_DX7_p3_4
ORACLE_PASSWORD=smhrd4          # ← 실제 비밀번호로 변경
ORACLE_HOST=project-db-campus.smhrd.com
ORACLE_PORT=1524
ORACLE_SERVICE_NAME=MAPPP
```

### 3단계: 환경 변수 확인

```powershell
python check_env.py
```

**또는 직접 확인**:
```powershell
python -c "import os; from dotenv import load_dotenv; from pathlib import Path; load_dotenv(Path('.env')); print('USER:', os.getenv('ORACLE_USER')); print('HOST:', os.getenv('ORACLE_HOST'))"
```

## ⚠️ 주의사항

- `.env` 파일은 `.gitignore`에 포함되어 있어야 합니다 (비밀번호 포함)
- 파일 이름이 `.env.txt`가 아닌 `.env`인지 확인
- Windows에서는 파일 이름 앞에 점이 있을 때 숨김 파일로 보일 수 있습니다
